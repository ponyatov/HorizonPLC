### DevicesManager

Для приема команд обрабатывает следующие события
```js
'sensor-start-polling', (_arg)      - запуск опроса с переданной частотой 
'sensor-stop-polling'               - его остановка
'sensor-get-info'                   - сбор метаданных данных о сенсорах
'sensor-write'                      - перенаправление команды актуатору
```

Для передачи данных другим службам вызывает следующие события
```js
'sensor-data', dataPackage          - { channel_id1: channel_value1, channel_id2: channel_value2, ... }
'sensor-info', infoArray            - возвращает - массив типа [{ID, _Name, Type, ...}, {...}, {...}]
```

```js
/**
 * @method
 * Выполняет инициализацию всех шин, указанных в конфиге к текущей программе.
 */
InitBuses()
/**
 * @method
 * Запускает периодичное считывание данных с сенсоров. Данные, которые не обновились с момента последнего обхода, пропускаются
 * @param {Number} _freq - частота опроса 
 */
StartPolling(_freq)
/**
 * @method
 * Прекращает периодический опрос датчиков
 */
StopPolling()
/**
 * @method
 * Собирает и возвращает информацию о датчиках.
 * Информация о том, какие поля требуется собрать, захардкожена в методе
 * @param {[String]} idArr - массив id
 */
GetSensorsInfo()
/**
 * @method
 * Инициализирует устройство по переданному id
 * Выполняет обращение к Process для получения конфига устройства
 * 
 * @param {Object} opts 
 * 
 * @returns {Object} Объект датчика
 */
CreateDevice(id, opts) 
```
